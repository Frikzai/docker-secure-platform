Analyse un faux fichier de logs SSH afin d’identifier une activité suspecte et retrouver le flag caché.

Difficulté {{ challenge.difficulty }} Statut : {{ challenge.status }}

Objectif

Un attaquant a tenté plusieurs connexions SSH avec différents utilisateurs. Ton objectif est d’analyser le fichier de logs et de trouver le flag.

Fichier à analyser

Télécharge ou ouvre le fichier suivant :

Ouvrir auth.log

Commandes utiles

grep "Failed password" auth.log
grep "FLAG" auth.log
awk '{print $1,$2,$3,$11}' auth.log
sort
uniq -c

Questions

  1. Quelle IP a effectué plusieurs tentatives de connexion SSH ?
  2. Quels noms d’utilisateurs ont été testés ?
  3. Quelle ligne contient le flag ?
  4. Quel est le flag complet ?

Validation

Entre le flag trouvé dans le fichier de logs.
